Hey Guys and Gals! Today we are going to be talking about Arrays and Hashes.

As we know, Ruby is a language full of objects. There are many ways to keep track of all of these objects, but the two most common ways are using Arrays and Hashes.

Arrays and hashes are used to house a collection of objects. These objects are indexed in arrays and hashes and can be called upon by using a key. This is very useful when you have a long list of objects that you need to store somewhere. Instead of having to go through each object, you can just use a key to call the exact object you want. It is kind of like a remote control. There are thousands of channels, but with the right key you can turn on any channel you want!

Ok Missy, this is all good, but when do we use an array and when do we use a hash?

Well that is a great question, and the answer is simple. Remember that key I was talking to (the numbers you click on the remote to call the channel you want)? Well when you are working with arrays, that key is an integer. When you are working with hashes that key can be any object. So on your television, you have to type in the integer 43 as the key to call MTV, this is like a hash. However if you are typing in the password -or key- for your Gmail account you can use letters, symbols, punctuation marks, or even integers to call your account, this is more like a hash.

I don’t know about you guys, but I learn better by seeing examples. So here are a few simple examples to help me illustrate hashes and arrays.

Here are some arrays:

array1 = ["Brooklyn", "Manhattan", "Jersey", "Queens"]

array2 = [5, 7, 9, 17, 48]

array3 = ["Melissa", "5", "blue", 6.82]

As you can see, arrays can have handle strings, integers, and floats. now if I want to access the ‘Manhattan’ element in the first array, I would call it using the key 1. This is because when accessing elements in arrays you use positioning. The first element is indexed as 0, the second is 1, the third is 2, and so one. Let’s do it together:

array1 [1]

See that wasn’t so hard was it? Alrighty, now that we’ve got arrays down, lets move on to hashes. Hashes are represented as key-value pairs, as seen below:

fave_animals = {

"dog" => "spot",

"cat" => "fluffy",

"fish" => "goldie",

"turtle" => "speedy"

}

Instead of having to use integers to call each value, hashes allow you to simply use the given key. For example, to find out my favorite fish, I just call

fave_animals ["fish"].

So that is it folks. Arrays and Hashes are very simple, you just have to remember where you put the key (and what it looks like)!
