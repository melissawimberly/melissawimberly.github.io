Hey Guys!

The blog for today is about the Enumerable .map. But first thing first; What is an Enumerable? Enumerables are a set of methods that provide shortcuts for manipulating collections of data. They are used as a sort of a proxy for each. There are several of these shortcut methods, such as all?, any?,sort, inject,min, max, and the one that we are going to be discussing which is map.

The map enumerable is a widely used enumerable because it modifies a collection of data, (usually that of an array or a hash), and transforms a copy of the grouped data without altering the original. It is essentially just like calling the each method on a group of data, except map does not alter the original data.

Ok, that was a lot of programming jargon and general bladiblah, how about we see this map enumerable in action!

some_array = [5,10,15,20]

new_array = some_array.map{ |x| -x}

puts some_array.join (',') #=> 5, 10, 15, 20

puts new_array.join (',') #=> -5, -10, -15, -20

Did you see that? The map method returned a copy of the original array with all of the numbers as negative numbers, but the original array is still in tact. But hang on! Why can’t we just use .each for this? Good question. The answer is because of the inheritant properties of *.each*. Let’s look at an example of what *.each* would look like for this example:

some_array = [5,10,15,20]

some_array.each { |x| x= -x}

puts some_array.join (',') #=> 5, 10, 15, 20

You see, in .each iteration, the assignment operation does not actually alter each element in the iteration. Rather, the x is a pointer to the current element in the collection. Changing x to point to another object does not change the actual element in the array. It just points the *puts* statement to something else, and that pointer is irrelevant at the end of the loop. So you are left with no actual change made.

There are a few quick notes that I want to point out:

If you do want to modify the original array, you can use map! (this is called map with a bang). Everything else is the same, but this is a destructive method, so the original array will be changed.

This .map enumerable can also be referred to as .collect.

Time for a recap:

Enumerables are a bunch of methods used as shortcuts to manipulate data. It is very similar to the .each method, except .map returna a copy of the data without affecting the original data.
